## 1. Mission & Vision

### Mission
**"분쟁 데이터를 검증 가능하고 접근 가능하게 만든다."**

### Vision
> GTI가 연 1회 하는 일을, 매일.
> Janes가 정부에게 하는 일을, 오픈소스로 모든 사람에게.

### 핵심 원칙
1. **투명성** — 데이터 출처, 신선도, 정확도를 숨기지 않는다
2. **측정 가능한 품질** — "좋은 것 같다"가 아니라 "정밀도 87%"
3. **데이터 > AI** — LLM은 3줄 요약만. 진짜 가치는 구조화 데이터
4. **365일 무중단** — 하루도 빠지지 않는 실행이 신뢰의 기반
5. **오픈 코어** — 핵심은 공개, 부가가치로 수익

## 2. 문제 정의

### 현재 시장의 구조적 문제

전 세계 분쟁/테러 인텔리전스 시장은 **두 극단**에 갇혀 있다:

```
무료/학술                           엔터프라이즈
(ACLED, UCDP, GTD)                 (Janes, Recorded Future, Dataminr)
- 정확하지만 2개월 지연              - 실시간이지만 $100K~500K/년
- UX 2010년대 수준                  - 정부/대기업만 접근
- API 제한적                        - 폐쇄적
- 교차검증 없음                      - 단일 벤더 lock-in

              ┌─────────────────┐
              │   $0~500/월     │
              │   아무것도 없음   │
              └─────────────────┘
```

### 누가 고통받는가

| 사용자 | 현재 상황 | 고통 |
|--------|----------|------|
| 소규모 보안 컨설턴트 | 뉴스 수동 모니터링 | 매일 2시간 소모, 놓치는 사건 多 |
| NGO 현장팀 | ACLED 수동 조회 | 1주 지연된 데이터로 안전 판단 |
| 프리랜서 리스크 분석가 | Janes 쓸 돈 없음 | 클라이언트에게 신뢰도 높은 데이터 제공 불가 |
| 대학 안보연구소 | GTD(2020 중단) | 최신 데이터 접근 불가 |
| 여행 리스크 스타트업 | 데이터 소스 구매 비용 | API 하나에 $50K/년 |
| 보험 언더라이터 | 수동 국가 리스크 평가 | 정량 데이터 없이 감으로 판단 |
| 저널리스트 | 여러 소스 수동 취합 | 취재 기획에 구조화 데이터 부재 |

### 카테고리별 상세

#### 연간 지수 발표 기관

| 이름 | 핵심 | 가격 | 약점 |
|------|------|------|------|
| **GTI (IEP)** | 163개국 테러 영향 지수 | 보고서 무료 | 연 1회, API 없음, "오늘" 못 답함 |
| **Fragile States Index** | 178개국 취약성 순위 | 무료 | 연 1회, 테러 특화 아님 |
| **GPI** | 163개국 평화 지수 | 무료 | 연 1회, 행동 지침 없음 |
| **INFORM** | 191개국 위기 리스크 | 무료 | 반기~연간, 실무 활용 어려움 |

#### 엔터프라이즈 플랫폼

| 이름 | 가격 | 킬러 피처 | 약점 |
|------|------|-----------|------|
| **Recorded Future** | $100K~500K/년 | AI 기반 위협 점수, Intelligence Cards | 사이버 중심, 테러 부차적 |
| **Janes JTIC** | $150K+/년 | 검증된 사건 DB, 조직 프로파일 | 폐쇄적 |
| **Dataminr** | $500K+/년 | 실시간 SNS 감지 (분 단위) | 극고가 |
| **SITE Intelligence** | 기관 구독 | 극단주의 플랫폼 직접 모니터링 | 접근 제한 |
| **Flashpoint** | $50K+/년 | "Sparks" 마이크로 업데이트, 딥웹 | PE 소유, 가격 상승 중 |
| **Dragonfly** | 기관 구독 | 70K+ 사건 DB, 사전 음모 추적 | 고가 |
| **Crisis24** | 기관 구독 | 여행자 위치 추적 + 위협 매칭 | 대기업 전용 |

#### 무료/학술

| 이름 | 강점 | 약점 |
|------|------|------|
| **ACLED** | 구조화, 주간 업데이트 | 상업 라이선스, 파생상품 금지 |
| **UCDP** | 골드 스탠다드 정확도, 사상자 | 2개월 지연 |
| **GTD** | 20만건 역사 | 2020년 중단 |
| **Liveuamap** | 실시간 분쟁 지도 | 구조화 데이터 없음 |
| **ViEWS** | 1~36개월 분쟁 예측 | 학술, 실무 UX 없음 |

### 경쟁사가 못하는 것 (시장 공백)

1. **실시간 + 구조화 통합** — 실시간(GDELT)과 구조화(UCDP)를 합친 것이 없음
2. **교차 소스 자동 검증** — 같은 사건을 여러 소스에서 확인하는 자동화 없음
3. **중간 가격대** — $0 ↔ $100K 사이 전문가급 도구 부재
4. **일일 변동 추적** — GTI는 연간, 가장 빠른 것도 주간(ACLED)
5. **개발자 친화적 API** — 합리적 가격의 테러/분쟁 데이터 API 없음
6. **임베드 가능한 위젯** — 뉴스사이트에 붙일 수 있는 분쟁 데이터 위젯 없음

### 5.3 API

```
Base URL: https://api.terror.david.dev/v1

GET /threats                    → 전체 국가 위협도 (Daily Score)
GET /threats/{country}          → 국가별 상세 (사건, 조직, 트렌드)
GET /threats/{country}/history  → 시계열 (일별/주별/월별)
GET /events                     → 오늘의 사건 (필터: source, country, date)
GET /events/{id}                → 사건 상세
GET /orgs                       → 활동 조직 목록
GET /orgs/{name}                → 조직 프로필 (활동 이력, 국가, 공격유형)
GET /hotspots                   → 현재 핫스팟 (좌표, 사건수, 사망자)
GET /sparks                     → 최근 마이크로 업데이트
GET /status                     → 소스별 수집 상태 + 데이터 신선도
```

## 6. 성장 전략

### Growth Loop (핵심 성장 엔진)

```
임베드 위젯 (뉴스사이트에 무료 배포)
  → "Powered by David" 클릭
    → 대시보드 발견
      → 관심 국가/조직 구독 (lock-in)
        → 매일 점수 변동 → 매일 방문 (리텐션)
          → 큰 사건 시 이벤트 페이지 공유 (바이럴)
            → SNS에서 새 사용자 유입
              → 반복
```

### 단계별 성장 전략

| 단계 | 전략 | 목표 |
|------|------|------|
| **0~3개월** | GitHub 오픈소스 + HN/Reddit 런칭 | Stars 100, 인지도 |
| **3~6개월** | 임베드 위젯 배포 시작, OSINT 커뮤니티 침투 | MAU 500 |
| **6~12개월** | API 출시, 대학/컨설턴트 온보딩 | MAU 2,000, API 50팀 |
| **12~24개월** | SaaS 전환, B2B 영업 시작 | MRR $5K |

## 8. 기술 아키텍처

### 현재 (구축 완료)

```
GitHub Actions (매일 UTC 06:00)
  → Python 파이프라인
    → 6소스 병렬 수집
    → 매칭/분석/위협도 계산
    → SQLite + Markdown 보고서
    → git commit & push
```

### 목표

```
┌─────────────────────────────────────────────────────┐
│  Data Pipeline (Python, GitHub Actions)              │
│  GDELT + UCDP + News + 17 RSS + Sanctions + OFAC    │
│  → Enrich → Link → Analyze → DB                    │
└──────────────────┬──────────────────────────────────┘
                   │ 매일 DB 업데이트
                   ▼
┌─────────────────────────────────────────────────────┐
│  API Server (Next.js API Routes / FastAPI)           │
│  /api/threats  /api/events  /api/orgs  /api/sparks  │
└───┬──────────┬──────────┬──────────┬────────────────┘
    │          │          │          │
    ▼          ▼          ▼          ▼
┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐
```

### 기술 스택

| 레이어 | 선택 | 이유 |
|--------|------|------|
| **프론트엔드** | Next.js 14+ (App Router) | SSR/SSG, API Routes 내장, Vercel 배포 |
| **지도** | Mapbox GL JS 또는 Leaflet | 핫스팟 히트맵, 인터랙티브 |

| **파이프라인** | Python + GitHub Actions | 현재 그대로 유지 |
