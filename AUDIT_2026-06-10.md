## 1. 🔴 가장 심각 — 자동 수집이 6주째 멈춤 (원인 규명 완료)

### 무슨 일이 있었나
- 마지막으로 데이터가 들어온 날: **2026-04-27**
- 오늘: 2026-06-10 → **약 6주간 신규 데이터 0건**
- 사이트는 "매일 자동 업데이트(Updated daily)"라고 안내 중 → 현재는 사실과 다름

### 진짜 원인 (확정)
매일 자동 실행을 담당하는 **Windows 예약 작업 `TerrorDailyReport`** 이 매일 실패하고 있었습니다.

- 실패 코드: **`0x8007010B` = "디렉터리 이름이 잘못되었습니다"**
- 예약 작업이 가리키는 경로: `C:\Users\강성준\Desktop\terror\...` ← **존재하지 않음**
- 실제 프로젝트 경로: `C:\Users\강성준\Desktop\app\terror\...` ← `\app\` 폴더가 빠짐

즉, **프로젝트 폴더를 `Desktop\terror` 에서 `Desktop\app\terror` 로 옮겼는데, 예약 작업은 옛 경로를 그대로 가리켜서** 매일 즉시 실패했습니다. (실행 파일 `run_daily.bat` 내부 경로는 이미 올바르므로, 예약 작업 등록 경로만 고치면 됩니다.)

### 👉 사용자가 직접 해야 할 1줄 (PowerShell 관리자 권한)
```powershell
$p = "C:\Users\강성준\Desktop\app\terror"
Set-ScheduledTask -TaskName "TerrorDailyReport" `
  -Action (New-ScheduledTaskAction -Execute "$p\scripts\run_daily.bat" -WorkingDirectory $p) `
  -Settings (New-ScheduledTaskSettingsSet -AllowStartIfOnBatteries -DontStopIfGoingOnBatteries -StartWhenAvailable)
```
> 보안상 자동 적용은 차단되어 직접 실행이 필요합니다.
> 적용 후 즉시 테스트: `Start-ScheduledTask -TaskName "TerrorDailyReport"`
> (참고: 노트북이 배터리 상태면 기존 설정상 실행이 안 됐는데, 위 명령이 그 제한도 함께 풀어줍니다.)

## 2. 🟠 오염 데이터 정리 (완료)

### (A) 분쟁과 무관한 노이즈 288건 삭제
RSS 전문가 피드(The War Zone, Military Times 등)가 관련성 필터를 면제받아,
분쟁과 무관한 기사가 "사건"으로 잘못 저장되고 있었습니다.

- 예: `NASA 연구 지원자 모집`, `항공모함 추적기`, `해군대학원 동문 소식`, `반도체 칩 개발`
- 삭제 기준: **국가 미상 + 사망자 0** 인 자동수집(gdelt/expert_rss/google_news) 행
- 삭제 결과: gdelt 228 + expert_rss 47 + google_news 13 = **288건 제거**

### (B) 국가 이름 분리(중복) 105건 통합
같은 나라가 ISO 코드와 정식명으로 따로 집계되고 있었습니다.

- 예: `NG`/`Nigeria`, `PK`/`Pakistan`, `UA`/`Ukraine`, `SD`/`Sudan` 가 별개 국가로 카운트
- 원인: GDELT 보강 과정에서 일부가 `NG`(코드) → `Nigeria`(이름) 변환에 실패
- 조치: 105건을 정식명으로 통합 → **국가 수 209개 → 167개** (정상화)

### 결과
| 항목 | 점검 전 | 점검 후 |
|------|--------|--------|
| 전체 사건 수 | 419,606 | **419,318** (노이즈 -288) |
| 집계 통계 기준 사건 수 | 419,498 | **419,210** |
| 고유 국가 수 | 209 (부풀림) | **167** |
| 국가 미상 행 | 288 | **0** |
| ISO코드가 국가명인 행 | 105 | **0** |

> 안전을 위해 정리 전 DB를 `data/terror.db.bak` 으로 백업했습니다. 확인 후 삭제하셔도 됩니다.

### 재발 방지 (코드 수정 완료)
- `scripts/sources.py` — GDELT 수집 시 **국가 미지정 이벤트는 저장하지 않음**
- `scripts/database.py` — 뉴스 저장 시 **국가 미상 + 사망자 0 인 비분쟁 기사는 저장하지 않음**

## 4. 🟢 "문제 같지만 문제 아님" — 오진 바로잡기

자동 분석 에이전트가 심각하다고 보고했으나, **직접 검증해 보니 사실이 아닌** 항목들입니다.

### (A) "전체 데이터의 40%(17만 건)가 중복" → ❌ 사실 아님
- 모든 사건의 고유 ID 419,318개가 **전부 다름** (중복 적재 없음)
- 같은 날짜·국가·사망자 행들은 UCDP의 **개별 사건**임 (서로 다른 마을·좌표: Ilovaisk, Komsomolskoye, Starobesheve…)
- 에이전트가 "날짜+국가+사망자 같으면 중복"이라는 잘못된 기준을 적용한 오진

### (B) "config의 모델명 `gpt-5.4-mini`는 가짜" → ❌ 사실 아님
- 최신 리포트(04-27)의 BLUF 요약이 **정상 생성**됨 (실패 시 "BLUF 생성 실패" 문구가 떠야 하나 안 뜸)
- 모델은 정상 작동 중 → **건드리지 않음**

### (C) "Ethiopia 12만 사망은 조작" → ⚠ 부분 오진
- 해당 7건은 모두 `is_aggregate=1`(집계 사건) 플래그가 걸려 있고, 통계는 이미 이들을 제외해 계산함
- 시스템이 정상 처리 중 (단, 화면에 집계 사건이 개별 사건처럼 안 보이는지는 추후 점검 권장)
